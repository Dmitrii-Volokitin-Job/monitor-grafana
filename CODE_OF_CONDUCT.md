# Code of Conduct

This project follows the
[Contributor Covenant 2.1](https://www.contributor-covenant.org/version/2/1/code_of_conduct/).
The short version: be welcoming, be respectful, and assume good intent.

## TL;DR

- **Be kind.** Treat reviewers, contributors, and users the way you'd want to
  be treated.
- **Give the benefit of the doubt.** Different time zones, languages, and
  experience levels make text-based communication harder than it looks.
- **Focus on the code, not the person.** "This function has a race condition"
  is feedback; "you can't code" is not.
- **Privacy matters.** Don't share other contributors' private info without
  their consent — including snippets of private DMs.

## Where this applies

In every project space: GitHub issues and PRs, code comments, any official
chat channels, and any in-person events organised under the project's name.

## Reporting

If you experience or witness behaviour that violates this code, email the
project maintainers privately. We will:

1. Acknowledge within 72 hours.
2. Investigate confidentially.
3. Decide on a response (warning, temporary ban, permanent ban) and explain
   it to the reporter and (where appropriate) the wider community.

Reporting in good faith will never be held against you. False or malicious
reports will be treated as a violation themselves.
