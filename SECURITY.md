# Security policy

See [`docs/SECURITY.md`](docs/SECURITY.md) for the full policy — vulnerability
reporting, supported-version table, pre-publish secret-handling checklist.

## TL;DR

- **Report a vulnerability** privately via GitHub's "Report a vulnerability"
  button on the [Security tab](https://github.com/Dmitrii-Volokitin-Job/monitor-grafana/security)
  of this repo. Please do NOT open a public issue for security reports.
- **Currently supported**: latest `v0.x.x` release on the `main` branch.
- **No bug bounty.** This is an open-source project maintained as time
  permits — acknowledgement in the changelog and a credit in the fix commit
  is all we can offer for responsible disclosures.

The full policy in `docs/SECURITY.md` also covers what we consider
in-scope vs out-of-scope, our triage timelines, and the pre-publish
checklist contributors should run before pushing a change that touches
credentials, env-var defaults, or the admin UI.
